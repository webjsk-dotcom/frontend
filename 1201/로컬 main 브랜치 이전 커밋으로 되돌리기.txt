1️⃣ 로컬 main 브랜치 이전 커밋으로 되돌리기
git checkout main
git reset --hard HEAD~1
•   HEAD~1 → 마지막 커밋 하나 이전
•   --hard 옵션 → 워킹 디렉토리와 스테이지까지 이전 상태로 변경
•   주의: 로컬 main의 마지막 커밋은 사라짐 (하지만 백업 브랜치를 만들어두면 안전)
________________________________________
2️⃣ 원격 main 강제 업데이트
git push origin main --force
•   로컬 main 상태를 그대로 원격 main에 덮어씀
•   원격 main의 마지막 커밋도 하나 이전 상태로 돌아감
•   주의: 다른 사람이 이미 pull 받은 커밋이 있다면 충돌 가능
