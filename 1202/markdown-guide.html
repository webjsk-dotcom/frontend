📝 마크다운(Markdown) 완전 가이드

마크다운(Markdown)은 텍스트 기반의 경량 마크업 언어로, 쉽고 빠르게 문서를 작성하고 HTML로 변환할 수 있습니다. 이 가이드에서는 마크다운의 기본 개념부터 모든 문법을 자세히 설명합니다.

1. 마크다운이란?

1.1 마크다운 소개

마크다운(Markdown)은 2004년 존 그루버(John Gruber)와 에런 스워츠(Aaron Swartz)가 만든 텍스트 기반의 경량 마크업 언어입니다.

마크다운은 일반 텍스트로 작성하되, 특정 문법을 사용하여 서식이 있는 문서로 변환할 수 있습니다. HTML보다 훨씬 간단하고 읽기 쉬우며, 다양한 플랫폼에서 지원됩니다.

1.2 마크다운의 특징

  • 간단한 문법: HTML보다 훨씬 쉽고 직관적
  • 읽기 쉬움: 원본 텍스트도 읽기 편함
  • 플랫폼 독립적: 텍스트 에디터면 어디서나 작성 가능
  • HTML 변환: 마크다운 문법이 HTML 태그로 변환됨
  • 버전 관리 용이: 텍스트 파일이므로 Git 등에서 차이점 추적 쉬움
💡 핵심 개념: 마크다운의 각 문법은 HTML 태그로 변환되어 브라우저에 출력됩니다. 따라서 눈에 보이는 스타일이 아닌 각 문법의 의미에 맞게 사용해야 합니다.

2. 장단점

✅ 장점

  • 문법이 쉽습니다.
    HTML보다 훨씬 간단한 문법으로 빠르게 문서 작성 가능
  • 관리가 간편합니다.
    텍스트 파일이므로 버전 관리와 백업이 쉬움
  • 지원 가능 플랫폼 및 프로그램이 다양합니다.
    거의 모든 텍스트 에디터와 많은 웹 서비스에서 지원

❌ 단점

  • 표준이 없어 사용자마다 작성 문법이 조금씩 다를 수 있습니다.
    CommonMark, GitHub Flavored Markdown 등 다양한 변형 존재
  • 모든 HTML 마크업을 대체하지 못합니다.
    복잡한 레이아웃이나 특수한 HTML 기능은 직접 작성 필요

3. 사용처

3.1 다양한 플랫폼에서 활용

마크다운은 메모장부터 전용 에디터까지 많은 곳에서 활용할 수 있습니다.

3.2 추천 에디터

  • VS Code: 마크다운 미리보기 기능 제공 (추천)
  • Typora: 실시간 미리보기 에디터
  • Obsidian: 마크다운 기반 노트 앱
  • Notion: 마크다운 문법 지원
  • 일반 텍스트 에디터: 메모장, Vim, Emacs 등

3.3 지원하는 서비스

  • 블로그: GitHub Pages, Jekyll, WordPress
  • 협업 도구: Slack, Notion, Discord
  • 코드 호스팅: GitHub, GitLab, Bitbucket (README.md)
  • 문서화: GitBook, Docusaurus, MkDocs
💡 팁: 문법이 쉬워 꼭 에디터를 사용할 필요는 없지만, VS Code 같은 코드 에디터에서 편집하는 것을 추천합니다. 미리보기 기능과 문법 하이라이팅이 제공됩니다.

3.4 스타일의 차이

마크다운의 각 문법은 HTML로 변환되어 CSS와 함께 출력되기 때문에, 화면에 표시되는 모양은 사용하는 곳의 스타일 구성에 따라 달라집니다.

예를 들어, 같은 # 제목 문법이라도:

  • GitHub에서는 큰 제목으로 표시
  • Notion에서는 중간 크기 제목으로 표시
  • 블로그 테마에 따라 다른 스타일로 표시

  2. 파일 확장자를 .md로 저장
  3. 마크다운 문법으로 작성

5. 문법 개요

5.1 기본 원리

각 마크다운 문법은 HTML 태그로 변환되어 브라우저에 출력됩니다.

모든 HTML 태그를 대체하지 않으며, 일부 중요한 문법만을 제공합니다. 또한 일부 문법만 지원하는 플랫폼도 있습니다.

⚠️ 주의: 플랫폼마다 지원하는 마크다운 문법이 다를 수 있습니다. 예를 들어, GitHub Flavored Markdown(GFM)은 테이블, 체크박스 등을 추가로 지원합니다.

5.2 주요 문법 목록

  • 제목 (Headers) - # 사용
  • 강조 (Emphasis) - *, **, ~~ 사용
  • 목록 (Lists) - -, 1. 사용
  • 링크 (Links) - [텍스트](URL) 형식
  • 이미지 (Images) - ![대체텍스트](URL) 형식
  • 코드 (Code) - `코드` 또는 ```코드블록```
  • 인용 (Blockquote) - > 사용
  • 수평선 (Horizontal Rule) - --- 사용

6. 제목 (Headers)

6.1 해시 기호(#) 사용

<h1>, <h2>, <h3>, <h4>, <h5>, <h6> 태그로 변환되는 '제목(Header)'을 표현합니다.

마크다운

# 제목 1

## 제목 2

### 제목 3

#### 제목 4

##### 제목 5

###### 제목 6

출력 결과

제목 1

제목 2

제목 3

제목 4

제목 5
제목 6

6.2 대체 문법 (H1, H2만)

<h1>, <h2>는 다음과 같이 표현할 수도 있습니다.

마크다운

제목 1
======

제목 2
------

출력 결과

제목 1

제목 2

⚠️ 주의: 책에서 목차는 여러 단계가 있지만 책 제목은 하나인 것처럼, 하나의 마크다운 파일에서는 하나의 <h1>(대제목)만 사용해야 합니다.

7. 강조 (Emphasis)

7.1 강조 문법

각각 <em>(기울임), <strong>(두꺼움), <del>(취소선) 태그로 변환되는 '강조(Emphasis)'를 표현합니다.

마크다운에서 지원하지 않는 밑줄을 추가하려면, <u></u>(밑줄) 태그를 직접 사용할 수 있습니다.

마크다운

이텔릭체는 *별 기호(Asterisks)* 혹은 _언더바 기호(Underscore)_ 를 사용하세요.

두껍게는 **별 기호(asterisks)** 혹은 __언더바 기호(underscore)__ 를 2번씩 사용하세요.

__*이텔릭체*와 두껍게__ 를 혼용할 수도 있습니다.

취소선은 ~~물결 기호(tilde)~~ 를 사용하세요.

<u>밑줄</u>은 마크다운에서 지원하지 않기에, 직접 `<u></u>` 태그를 사용하세요.

출력 결과

이텔릭체는 별 기호(Asterisks) 혹은 언더바 기호(Underscore) 를 사용하세요.

두껍게는 별 기호(asterisks) 혹은 언더바 기호(underscore) 를 2번씩 사용하세요.

이텔릭체와 두껍게 를 혼용할 수도 있습니다.

취소선은 물결 기호(tilde) 를 사용하세요.

밑줄은 마크다운에서 지원하지 않기에, 직접 <u></u> 태그를 사용하세요.

7.2 강조 문법 요약

효과 문법 HTML 태그
기울임 *텍스트* 또는 _텍스트_ <em>
두껍게 **텍스트** 또는 __텍스트__ <strong>
취소선 ~~텍스트~~ <del>
밑줄 <u>텍스트</u> <u> (직접 HTML 사용)
⚠️ 주의: '이텔릭체와 두껍게 를' 과 같이 한글에서는 강조 표현 다음에 띄어쓰기가 필요할 수 있습니다.

8. 목록 (Lists)

8.1 목록 문법

<ol>, <ul>, <li> 태그로 변환되는 '목록(List)'을 표현합니다.

  • 1. 로 시작하는 항목을 작성하면 <ol>로 변환됩니다.
  • -로 시작하는 항목을 작성하면 <ul>로 변환됩니다.
  • 들여쓰기를 통해 하위 목록을 작성할 수 있습니다.

마크다운

1. 순서가 있는 항목
1. 순서가 있는 항목
    1. 순서가 없는 항목
    1. 순서가 없는 항목
1. 순서가 있는 항목
1. 순서가 있는 항목

- 순서가 없는 항목
- 순서가 없는 항목
    - 순서가 없는 항목
    - 순서가 없는 항목

출력 결과

  1. 순서가 있는 항목
  2. 순서가 있는 항목
    • 순서가 없는 항목
    • 순서가 없는 항목
  3. 순서가 있는 항목
  4. 순서가 있는 항목
  • 순서가 없는 항목
  • 순서가 없는 항목
    • 순서가 없는 항목
    • 순서가 없는 항목

8.2 목록 문법 요약

타입 문법 HTML 태그
순서 있는 목록 1. 항목 <ol><li>
순서 없는 목록 - 항목 또는 * 항목 <ul><li>
⚠️ 주의: 하위 목록을 작성할 때, 들여쓰기(Indent)가 띄어쓰기 2칸이 아닌 3칸 이상일 수 있습니다.
💡 팁: 순서가 있거나 없는 목록을 혼합하여 사용할 수 있지만, 일부 플랫폼에서는 동작하지 않을 수 있습니다.

9. 링크 (Links)

9.1 링크 문법

<a>로 변환되는 '링크(Links)'를 표현합니다.

⚠️ 주의: '링크'는 현재 탭에서 열리는 것이 기본이지만, 새 탭에서 열릴 수도 있습니다.

11. 팁과 주의사항

11.1 마크다운 작성 팁

  • 일관성 유지: 같은 문서 내에서 문법 스타일을 일관되게 사용
  • 의미에 맞게 사용: 스타일이 아닌 의미에 맞게 문법 선택
  • 빈 줄 활용: 가독성을 위해 적절한 빈 줄 사용
  • 대체 텍스트 작성: 이미지에 반드시 대체 텍스트 추가

11.2 플랫폼별 차이점

⚠️ 주의: 플랫폼마다 지원하는 마크다운 문법이 다릅니다.
  • CommonMark: 표준 마크다운
  • GitHub Flavored Markdown (GFM): 테이블, 체크박스, 취소선 등 추가
  • Notion: 일부 문법만 지원
  • Slack: 매우 제한적인 문법 지원

11.3 일반적인 실수

  • 제목에 공백 없음: #제목 (X) → # 제목 (O)
  • 목록 들여쓰기 오류: 하위 목록은 3칸 이상 들여쓰기
  • 강조 문법 오류: *텍스트* 사이에 공백 없어야 함
  • 링크 괄호 오류: [텍스트](URL) 형식 정확히 지키기

11.4 추가 문법 (일부 플랫폼)

문법 설명 지원 플랫폼
```코드블록``` 코드 블록 대부분
> 인용 인용문 대부분
--- 수평선 대부분
